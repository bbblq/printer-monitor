# 飞书推送功能使用指南

## 快速开始

### 1. 配置飞书机器人

1. 在飞书群中添加自定义机器人
2. 获取Webhook URL（格式：`https://open.feishu.cn/open-apis/bot/v2/hook/xxxxx`）
3. 在系统管理面板中添加通知设置

### 2. 添加通知配置

进入管理面板 → 通知设置 → 添加新配置：

- **名称**: 例如 "飞书-运维群"
- **类型**: 选择 "飞书"
- **Webhook URL**: 粘贴你的机器人webhook地址
- **启用**: 勾选

### 3. 配置报表推送

在通知设置中配置：

#### 警报设置
- ✅ **耗材不足警报**: 当耗材低于设定百分比时推送
- ✅ **耗材耗尽警报**: 当耗材耗尽时立即推送
- ✅ **耗材更换通知**: 检测到更换耗材时推送

#### 定时报表
- ✅ **启用定时报表**: 勾选
- ✅ **Cron表达式**: 设置推送时间

### Cron表达式示例

```
格式: 分 时 日 月 周

0 9 * * 1      # 每周一上午9:00
0 18 * * *     # 每天下午6:00
30 8 * * 1-5   # 工作日上午8:30
0 9,18 * * *   # 每天上午9:00和下午6:00
0 9 1 * *      # 每月1号上午9:00
```

## 消息类型

### 📊 定时报表
- 蓝色卡片主题
- 包含内容：
  - 设备状态统计（总计/在线/离线）
  - 本月耗材更换记录
  - 全年耗材更换记录（最多显示10条）
  - 耗材不足警告

### 🚨 耗材耗尽警报
- 红色卡片主题
- 立即推送
- 包含打印机详细信息

### ⚠️ 耗材不足警报
- 橙色卡片主题
- 当耗材低于设定阈值时推送
- 包含打印机详细信息

### 🔄 耗材更换通知
- 绿色卡片主题
- 检测到耗材更换时推送
- 显示更换前后的耗材量

## 立即发送测试报表

### 方法1: 通过管理面板
1. 进入管理面板
2. 点击"通知设置"
3. 点击"立即发送报表"按钮

### 方法2: 通过API
```bash
curl -X POST http://your-server/api/admin/notifications \
  -H "Content-Type: application/json" \
  -d '{"action": "send_report"}'
```

### 方法3: 使用测试脚本
```bash
# 测试报表消息
node test-report.js

# 测试警报消息
node test-alerts.js
```

## 故障排查

### 消息发送失败

1. **检查Webhook URL**
   - 确保URL格式正确
   - 确保机器人未被移除

2. **查看发送历史**
   - 进入管理面板 → 通知历史
   - 查看错误信息

3. **测试连接**
   ```bash
   node test-feishu.js
   ```

### 定时报表未发送

1. **检查Cron表达式**
   - 确保格式正确（5个字段）
   - 确保时间设置合理

2. **检查服务状态**
   - 确保应用正在运行
   - 检查日志中的 `[Report Scheduler]` 信息

3. **检查通知设置**
   - 确保"启用定时报表"已勾选
   - 确保通知配置本身已启用

## 最佳实践

### 1. 合理设置推送频率
- 报表：建议每天或每周推送一次
- 警报：保持实时推送

### 2. 设置合适的阈值
- 耗材不足：建议10-15%
- 根据实际使用情况调整

### 3. 多渠道配置
- 可以同时配置飞书、企业微信、邮件
- 重要警报使用多个渠道

### 4. 定期检查
- 每周检查通知历史
- 确保消息正常发送
